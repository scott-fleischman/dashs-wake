# Repository Instructions

## Published Build

- GitHub Pages serves the realized application from the committed `docs/`
  directory on the publishing branch.
- Before every commit, run `npm run verify` and stage all regenerated `docs/`
  changes in that same commit. This applies to tests, source, content,
  configuration, and documentation commits.
- Treat a commit without an up-to-date `docs/` tree as incomplete.
- Do not edit `docs/` by hand; it is generated by the Vite build.

## Verification Before Commit

`npm run build` only checks TypeScript and bundling. It does **not** load the
app in a browser and will miss runtime module-init failures (for example a
circular import that leaves `firstWakeLevel` undefined and produces a blank
screen).

Always run:

```bash
npm run verify
```

This runs, in order:

1. `npm run build` — typecheck and regenerate `docs/`
2. `npm run test:bootstrap` — import every official level module at runtime
3. `npm run test:smoke` — Playwright check that the default route renders UI

For gameplay or level-content changes, also run the full suites:

```bash
npm test
npm run test:e2e
```

## Level Content Architecture

Official level builders must avoid circular runtime imports.

- Put shared physics in `src/content/level-rules.ts` (`OFFICIAL_LEVEL_RULES`).
- `src/content/epic-course-builder.ts` may import types from `first-wake.ts`,
  but must **not** import runtime values such as `firstWakeLevel`.
- `first-wake.ts` initializes `firstWakeLevel` by calling `buildEpicLevel`.
  If `buildEpicLevel` reads `firstWakeLevel` during that initialization, the
  app crashes before the first frame renders.

When adding a new content module, extend `tests/unit/module-bootstrap.test.ts`
if it introduces a new entry point that must load at app startup.

## Agent Checklist

Before claiming a change is complete or creating a commit:

1. Run `npm run verify` and confirm it passes.
2. If you changed level layout, generator logic, or routing, run `npm test`.
3. If you changed UI boot, lobby, or level select, run
   `npm run test:e2e tests/e2e/app-boot.spec.ts` at minimum.
4. Do not treat `tsc --noEmit` or `vite build` alone as sufficient runtime
   validation.
