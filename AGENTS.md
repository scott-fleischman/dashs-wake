# Repository Instructions

## Published Build

- GitHub Pages serves the realized application from the committed `docs/`
  directory on the publishing branch.
- Before every commit, run `npm run verify:agent` and stage all regenerated
  `docs/` changes in that same commit. This applies to tests, source, content,
  configuration, and documentation commits.
- Treat a commit without an up-to-date `docs/` tree as incomplete.
- Do not edit `docs/` by hand; it is generated by the Vite build.

## Agent Verification (Required)

Agents must treat local verification as the release gate. Remote GitHub checks
are not a substitute for running verification in the workspace after changes.

After **any** source, test, or configuration change — and before claiming the
work is complete or creating a commit — run:

```bash
npm run verify:agent
```

This is the single command agents should use. It prints each step, fails fast,
and exits non-zero when verification fails.

### What `verify:agent` checks

| Step | Command | Catches |
|------|---------|---------|
| Build | `npm run build` | Type errors; broken imports that tsc catches |
| Agent gate | `npm run test:agent` | Circular module-init crashes; all 12 official levels failing to load; missing spawn support; generator boot failures |
| Browser smoke | `npm run test:smoke` | Blank screen at boot; level select not rendering; First Wake failing to start or advance |

Equivalent manual run:

```bash
npm run verify
```

### What is not enough

- `npm run build` alone — bundles successfully even when modules throw at import
  time (blank screen).
- `tsc --noEmit` alone — same limitation as build without browser smoke.
- Skipping Playwright smoke — misses UI boot and gameplay start regressions.

### When to run more

- **Level layout, generator, or simulation changes:** also run `npm test` and
  fix or extend the relevant unit tests.
- **Lobby, routing, pause, or full journey changes:** also run `npm run test:e2e`.
- **New startup entry points:** extend `tests/unit/agent-gate.test.ts`.

### Agent completion rule

Do not report success until `npm run verify:agent` exits 0 in the current
workspace. If it fails, fix the failure or report the blocker with command
output — never assume GitHub or a future CI run will catch it.

## Level Content Architecture

Official level builders must avoid circular runtime imports.

- Put shared physics in `src/content/level-rules.ts` (`OFFICIAL_LEVEL_RULES`).
- `src/content/epic-course-builder.ts` may import types from `first-wake.ts`,
  but must **not** import runtime values such as `firstWakeLevel`.
- `first-wake.ts` initializes `firstWakeLevel` by calling `buildEpicLevel`.
  If `buildEpicLevel` reads `firstWakeLevel` during that initialization, the
  app crashes before the first frame renders.

When adding a new content module that loads at app startup, extend
`tests/unit/agent-gate.test.ts`.
